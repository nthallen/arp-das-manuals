0.0: Invocation

    osupdate [options] <server node> <target node>
             Update destination node from server node
             Looks for distribution information in //$src/etc/config/osupdate.$src
             -f <file> Base configuration file name. Actual file will be
               //$src/etc/config/$file.$src  Default is osupdate.
             -R Delete target directory trees when a soft-link is specified
                (Use with caution!)
             -D Debug mode: don't actually do anything.
    
             Codes within the configuration file include:
               O: OS Installation: OS, TCP/IP, rundos
               O: OS, TCP/IP User, rundos (separated by config file)
               U: Upgrade (for temporary inclusion in primary file)
               Q: Server files (requires 'q' exclusion also)
               C: C Compiler Executables
               c: C Compiler Executables Soft-Linked
               L: Libraries and Include Files
                  (including /usr/local/include & lib)
               l: Libraries and Include Files soft-linked
            (requires the 'l' exclusion also)
               S: Watcom SQL
               W: Windows RT
               w: Windows Soft-Linked
               K: Windows TK
               k: Windows tk Soft-Linked
               T: TCP/IP Server
               B: /usr/local/bin
               b: /usr/local/bin soft-linked
               E: /usr/local/bin32
               e: /usr/local/bin32 soft-linked
               M: /usr/GNU/Emacs/emacs-18.55-qnx
               m: /usr/GNU/Emacs/emacs-18.55-qnx soft-linked
               F: Free Unix Utilities
               f: Free Unix Utilities soft-linked
               H: Documentation directories: man and doc
               h: Documentation directories soft-linked
               -: Obsolete files: delete if found
       Exclusions:
         n: Norton's System
         e: Eileen's System
         q: A Server ('Q' config code)
         l: A node with the 'l' config code
         f: A flight computer (not implemented)
             Flags after files
               S: File is to be soft-linked to source node file (may be specified)
               s: File is to be soft-linked to specified target node file
               H: File is to be hard-linked to specified target node file
